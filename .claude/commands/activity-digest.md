アクティビティログのダイジェストを表示する。

> **モデル運用**: この集計・整形は機械的処理のため、`report-collector`（Haiku）サブエージェントに委譲してよい（メインが Opus/Fable のときはトークン節約のため委譲を推奨）。事実の列挙のみを担わせ、評価・次アクションの提案はメイン側で行う。

引数: $ARGUMENTS（期間指定。例: "today", "7d", "30d"。省略時は "7d"）

以下のMCPツールを実行し、結果をエンティティ別に集計して出力せよ：

1. 引数から期間を計算:
   - "today" → 今日の00:00からISO8601
   - "7d" → 7日前のISO8601
   - "30d" → 30日前のISO8601
2. `activity_search` (from=計算した日時) でログを取得
3. entity_table別にグループ化して集計

出力フォーマット：

```
## アクティビティダイジェスト (期間: XX)

### エンティティ別集計
| テーブル | insert | update | delete | 合計 |
|---------|--------|--------|--------|------|
| tasks | X | X | X | X |
| meetings | X | X | X | X |
| ... | | | | |

### 最新アクティビティ (上位10件)
| 日時 | テーブル | アクション | 表示名 |
|------|---------|-----------|--------|
| HH:MM | entity_table | action | entity_display |
```
