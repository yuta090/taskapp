## 2. Problem — 「本業じゃないこと、多すぎませんか？」

### セクションタイトル
```
Problem

プロジェクトに関わる全員が、
「本業じゃないこと」に追われている。
```

### ペルソナ別の痛み（カード形式、3列 or スクロール）

**開発者の場合:**
- 「タスクの更新、管理画面を開いて、ステータス変えて、コメント書いて...コード書く時間が削られる」
- 吹き出し: 「Backlogの更新忘れてませんか？」

**PMの場合:**
- 「エンジニアの進捗をヒアリング → Excelに転記 → クライアントに報告。私は転記ロボットか？」
- 吹き出し: 「週次報告書、今週もですか...」

**クライアントの場合:**
- 「今どこまで進んでるの？聞かないとわからない。承認したはずなのに記録がない」
- 吹き出し: 「あの件、どうなりました？」

**代理店の場合:**
- 「制作会社の見積もりをExcelに転記、マージン計算して、クライアント用に作り直す。全部手作業」
- 吹き出し: 「原価率、合ってるよね...？」

## 3. Solution — 「1つのツールで、全員の"余計"が消える」

### セクションタイトル
```
Solution

AgentPMなら、1つのツールで
全員の余計な仕事が消える。
```

### 連鎖図（フロー形式）

```
[開発者]                    [PM]                      [クライアント]
  AIでタスク完了報告          ↓ 自動で反映               ↓ ポータルに即反映

```

### 補足コピー
```
開発者がコードをマージするだけで、PMの報告書が自動で更新され、
クライアントのポータルに進捗が反映される。
誰も転記しない。誰も催促しない。全員が本業に集中できる。
```

## 4. Feature: AI/MCP — 「管理画面を、もう開かなくていい」

### セクションタイトル
```
For Developers

普段使うAIが、
プロジェクト管理ツールになる。
```

### 本文
```
Claude、ChatGPT、Gemini——
いつものAIやターミナルからタスクの作成・更新・完了報告ができます。
ブラウザを開いてログインして管理画面で操作する必要はもうありません。
```

### 特徴リスト
- **AIからタスク作成・更新**: 「auth_v2.mdを読んでタスク分割して」で完了
- **GitHubマージで自動消し込み**: PRマージ → タスク完了 → クライアントに確認依頼が自動送信
- **見積もりも自然言語で**: 「追加機能の見積もり出して」→ 工数計算 → ポータルで承認依頼
- **CLIからもフルアクセス**: MCP対応の全アプリから操作可能

### ビジュアル
既存のFeatureTerminal（チャットアニメーション）を流用
+ GitHubマージ → タスク自動更新のフロー図を追加

## 5. Feature: Portal — 「URLを送るだけ。アカウント登録不要」

### セクションタイトル
```
For Clients

進捗が見える。だから聞かなくていい。
```

### 本文
```
クライアントにはポータル専用のURLを共有するだけ。
アカウント登録もアプリのインストールも不要です。
プロジェクトの進捗、承認待ちのアクション、スケジュールが一目で確認できます。
```

### 特徴リスト
- **アカウント登録不要**: URLを開くだけ。ITリテラシーを問わない
- **進捗が一目瞭然**: 「今何%？」を聞かなくても見える
- **承認はワンクリック**: 見積もり・デザイン・仕様の承認がポータル上で完結
- **証跡が残る**: 誰が・いつ・何を承認したか記録。言った言わない問題ゼロ

### ビジュアル
既存のFeaturePortal（ポータルUIモック）を流用

## 6. Feature: Ball — 「誰待ちか、もう聞かなくていい」

### セクションタイトル
```
For Everyone

すべてのタスクに「ボール」がある。
```

### 本文
```
「クライアント確認待ち」ならボールは相手に。
「修正作業中」ならボールは自分に。
責任の所在が常に明確だから、催促も確認も不要になります。
```

### 特徴リスト
- **ボール = 次に動くべき人**: タスクごとに「誰のターン」かが一目瞭然
- **自動でボールが移動**: GitHubマージ → ボールがクライアントへ。承認 → ボールが開発者へ
- **滞留アラート**: ボールが止まっている時間が長いと自動通知

### ビジュアル
既存のFeatureBall（アニメーション）を流用

## 7. For Agency — 「3者の間をシンプルに」

### セクションタイトル
```
For Agencies

クライアントと制作会社の"間"を、一画面に。
```

### 本文
```
代理店モードを有効にすると、原価・マージン・売値の3層管理が可能に。
制作会社は原価を入力、代理店はマージンを設定、クライアントには売値だけが見える。
Excel管理から卒業できます。
```

### 特徴リスト
- **原価/マージン/売値の一元管理**: Excelの転記ミスから解放
- **制作会社の進捗をそのまま共有**: 見せたい情報だけクライアントに表示
- **ロール別の権限管理**: 代理店・制作会社・クライアントそれぞれに適切な情報だけ

### ビジュアル
3者の関係図: [制作会社] → 原価 → [代理店] → マージン設定 → [クライアント] ← 売値のみ

### デザイン方針
このセクションは大きく取りすぎない。「こういう使い方もできます」程度のサイズ感。
興味ある人は /features や /for/agency に誘導。

## 8. Quick Start — 「5分で始められる」

### セクションタイトル
```
Getting Started

5分で、プロのPM環境が整う。
```

### ステップ
1. **アカウント作成** → メールアドレスだけで即開始
2. **プロジェクト作成** → テンプレートから選ぶだけ。受託開発、Web制作、アプリ開発etc
3. **AIでタスク登録** → 既存の仕様書やBacklogのCSVをAIに渡すだけで移行完了
4. **ポータルURL共有** → クライアントに送るだけ

### 補足コピー
```
既存ツールからの移行もAIにお任せ。
BacklogのCSV、GitHub Issues、スプレッドシートの内容をAIに渡すだけで
タスクが自動で登録されます。
```
