### 具体コマンド例
```bash
# タスクを自然言語で作成
> 「auth_v2.mdを読んで、ログイン機能のタスクを分割して登録して」

```

## 5. Feature: Portal — 「URLを送るだけ」

### セクションタイトル（id="for-clients"）
```
For Clients

進捗が見える。だから聞かなくていい。
```

### 本文（非IT層向けに平易に）
```
クライアントにはポータルのURLをお送りするだけ。
アプリのインストールも、アカウントの作成も必要ありません。
ブラウザを開いて、プロジェクトの今を確認し、承認ボタンを押す。
それだけで「確認しました」の連絡は不要になります。
```

### 特徴リスト
- **アカウント登録不要**: URLを開くだけ。ITに詳しくなくても大丈夫です
- **進捗が一目瞭然**: 「今何%完了しているか」がいつでも確認できます
- **承認はワンクリック**: 見積もり・デザインの承認がその場で完了します
- **証跡が残る**: 誰が・いつ・何を承認したか、全て記録されます

### セキュリティ
```
安心して使えるセキュリティ:
- ポータルURLは期限付き・個別発行。第三者はアクセスできません
- 表示する情報はプロジェクト管理者が選択できます
- 承認履歴はCSV/PDFでエクスポート可能。監査対応にも
- IP制限設定可能（Businessプラン）
```

#### [画像生成指示: Portal Feature]
```
左側: スマートフォンを持つビジネスパーソン（クライアント）が笑顔。
画面にはシンプルなポータル: 進捗バー82%、「承認する」ボタン。
右側: ポータル画面のPC表示。マイルストーン一覧、承認待ちリスト。
全体のトーン: シンプル、クリーン、「ITに詳しくなくても使える」安心感。
専門用語やコードは一切なし。
```


## 7. For Agency — 「3者の間をシンプルに」

### セクションタイトル（id="for-agencies"）
```
For Agencies

クライアントと制作会社の"間"を、一画面で管理。
```

### 本文
```
代理店モードなら、原価・マージン・売値の3層管理が一画面で完結。
制作会社は原価を入力。代理店はマージンを設定。
クライアントには売値だけが見える。
Excelでの転記・計算ミスから解放されます。
```

### 数字フロー（図）
```
[制作会社]                   [代理店]                    [クライアント]
 工数: 16h                    マージン率: 35%              表示金額:
 単価: ¥8,000/h        →     原価: ¥128,000        →     ¥172,800
 原価: ¥128,000               売値: ¥172,800               (原価は非表示)

 進捗: 60%完了          →     そのまま転送            →     60%完了
 (制作会社が更新)              (転記不要)                   (ポータルで確認)
```

### 特徴リスト
- **原価/マージン/売値の一元管理**: 案件ごとの粗利率がリアルタイム把握
- **案件横断の粗利ダッシュボード**: 全案件の収益性を一覧で確認
- **ロール別の権限管理**: 制作会社・クライアントに見える情報を個別設定
- **会計ソフト連携**: freee、マネーフォワード、弥生にCSV出力対応
- **粗利週次レポート自動送信**: 経営層への報告も自動化

### 信頼データ
```
導入前: 月末請求 = 案件あたり2h × 10案件 = 20時間
導入後: 案件あたり15分 × 10案件 = 2.5時間（-87%）
```
