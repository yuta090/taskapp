# LP修正案 v7 — Hero具体化 + 発注者メリット強化版

## v6からの変更点
- **Hero H1を具体化**: 抽象的な「全員が、本業に戻れる。」→ 便益が即わかるコピーに変更
- **発注者（中村）向けメリットを大幅強化**: Hero直下 + Portal セクション両方で発注者視点を追加
- **発注者の3大メリットを明確化**:
  1. ダッシュボードでカスタマイズされた状況把握ができる
  2. バグ報告・新タスクを統一UIから指示でき、自分たちも進捗を把握できる
  3. 見積もりも同じUIからすぐ受け取れて、歩留まりが少ない
- **仕様書・承認証跡の一体管理を訴求ポイントに追加**: Wiki・議事録・レビュー承認が同じツール内に残る強みを明記
- その他: レビュー指摘の「5分でスタート」の意味明確化、testimonial属性強化

## 3. Solution（v6からほぼ変更なし）

### タイトル
```
AgentPMなら、1つのツールで
全員の余計な仕事が消える。
```

### 連鎖フロー
```
[開発者]                   [PM]                     [発注者]
 AIでタスク完了         →   自動で進捗更新          →  ポータルに即反映
 GitHubマージで         →   転記不要                →  ワンクリック承認
 ボールが発注者へ                                      ↓ 承認通知がPMに届く

                           [代理店]
                            原価→マージン→売値を一画面管理
```

### コピー
```
開発者がタスクを完了すれば、発注者のポータルに即反映。
発注者がワンクリックで承認すれば、PMに通知が届く。
誰も転記しない。誰も催促しない。
```

### 他ツールとの違い（正直に）
```
AIやMCPへの対応は各ツールで進んでいます。

Backlog: AIアシスタント搭載、MCP対応。ただしクライアント共有にはアカウント作成が必要
Jira: Rovo AIは強力。ただし導入が複雑で、クライアント向けポータルは別製品
Linear: 開発者体験は最高。ただし日本語UI未対応、クライアント向け機能なし

AgentPMの違いは、CLIスキルで「進捗確認→レビュー→日程調整→議事録」が
ワークフローとして一気に回り、その結果がポータル・ボール管理まで自動で連鎖すること。
さらに、Wiki・議事録・レビュー承認がプロジェクト内に一体で残るため、
仕様の経緯と承認証跡が散逸しません。
他ツールもAI/MCPに対応し始めていますが、プロジェクトの立ち上げ自体が重ければ意味がありません。
AgentPMはテンプレート選択 → AIでタスク流し込み → ポータルURL共有で、
その日からプロジェクトが回り始めます。
「全員が繋がり、記録が残り、圧倒的に速く始められる」——これが本当の差別化です。
```

```

amber-500の矢印。「1つのデータ、全員に届く」。
```

## 4. Feature: AI/CLI（id="for-developers"）★v7変更: MCP→CLI/スキル中心に

### タイトル
```
For Developers
AIに話すだけで、プロジェクト管理が回る。
```

### コピー
```
Claude Code やターミナルから、自然言語でタスク管理を操作できます。
スキル（ワークフロー）を呼ぶだけで、
進捗確認・レビュー依頼・日程調整・議事録作成が一連で完了。
管理画面を開く必要はありません。
```

### スキル（ワークフロー自動化）★v7追加
```
スキルを使うと、複数の操作がワークフローとして一気に実行されます。

/project-status  → ダッシュボード・マイルストーン・ボール状態を横断取得してレポート出力
/meeting-flow    → 会議作成→開始→議事録記録→終了→アクションアイテムからタスク自動起票
/review-cycle    → レビュー依頼→承認/差し戻し→ボール移動まで一連で実行
/scheduling-wizard → Googleカレンダーから空き時間提案→日程調整→確定まで自動
/client-onboarding → クライアント招待→スペース追加→初期タスク作成を一括実行
```

### CLIコマンド例
```
# タスクを自然言語で作成・分割
> 「auth_v2.mdを読んで、ログイン機能のタスクを4つに分割して」
→ agentpm task create が4回実行され、マイルストーンも自動設定

# タスク完了 → ボールが自動で発注者へ
> agentpm task update --status done → agentpm ball pass --ball client
→ ポータルに通知が届く

# 見積もりを即時作成
> 「追加機能の見積もり、単価1万円で」→ ¥160,000
```

### 安全策
```
✓ 操作履歴からワンクリック復元
✓ 権限設定でAI操作を制限可能
✓ 全操作が監査ログに記録
```

### 対応コマンド
```
task（作成/更新/一覧/完了）/ ball（パス/照会）/ milestone / review /
meeting / minutes / wiki / scheduling / client / space / dashboard / activity
```
※ 裏側はMCPプロトコルで接続。Claude Code、Cursor等のMCP対応エディタからも操作可能

```

左: ターミナルで「/project-status」実行。レポートが出力される様子。
右: Web画面にリアルタイム反映。ボールラベルが「internal→client」に変わる瞬間。
```

## 5. Feature: Portal（id="for-clients"）★v7大幅強化

### タイトル
```
For Clients
聞かなくても、見ればわかる。指示もここから。
```
※ v6「URLを送るだけ。アカウント登録不要。」→ 発注者の行動ベネフィットに変更

### コピー
```
発注者専用のポータルで、プロジェクトの「今」がいつでも見えます。
アカウント登録もアプリも不要——URLを開くだけ。
```

### 発注者の3つのメリット（★v7追加）

```
1. 状況がひと目でわかる
   カスタマイズされたダッシュボードで、タスクの進捗・
   フェーズ・スケジュールがリアルタイムに表示されます。
   「今どうなってる？」をチャットで聞く必要がなくなります。

2. 指示がそのまま伝わる
   バグ報告や追加要望をポータルから直接起票できます。
   チャットに埋もれることなく、対応状況もそのまま追えます。
   開発チームとの「言った言わない」がゼロに。

3. 見積もりの歩留まりゼロ
   追加開発の見積もりはポータルに届きます。
   内容を確認して、ワンクリックで承認。
   メールやExcelのやり取りが不要で、承認の遅延がなくなります。

4. 仕様も承認も、証跡がぜんぶ残る ★v7追加
   Wiki・議事録・レビュー承認がプロジェクト内に一体で保存されます。
   「なぜこの仕様になったか」「誰がいつ承認したか」がいつでも辿れる。
   言った言わない問題だけでなく、仕様変更の経緯も完全に残ります。
```

### ポータルでできること（実装済み13画面ベース）
```
✓ プロジェクト進捗ダッシュボード（カスタマイズ可能）
✓ タスク一覧・詳細の確認
✓ バグ報告・追加要望の起票（★v7追加: 発注者からの起票）
✓ 承認依頼への対応（ワンクリック）
✓ ファイルの閲覧・ダウンロード
✓ 会議・議事録の確認
✓ Wiki（ナレッジ共有）
✓ 日程調整
✓ 操作履歴の確認
```

### セキュリティ
```
✓ URLはトークンベースで個別発行（推測不可能）
✓ 表示セクションはプロジェクト管理者が選択可能
✓ 承認・閲覧の全履歴を記録（監査対応）
✓ Supabase基盤: TLS暗号化通信、RLSによる行レベルセキュリティ
```

### Backlogとの違い
```
Backlog: クライアントも「ゲストユーザー」としてアカウント作成が必要。管理画面はエンジニア向けUI
AgentPM: URLを開くだけ。発注者に必要な情報だけが、わかりやすいUIで表示される
```

```

サブ: バグ報告フォーム画面（タイトル入力中）、見積もり承認画面（¥120,000 + 承認ボタン）
```
