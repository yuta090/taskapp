## A. 活用シーン (/use-cases)

### ページ目的
「自分のチームでどう使えるか」をペルソナ×業種で具体的にイメージさせる。
トップLPが「何ができるか」なら、活用シーンは「どう使うか」。

### ヒーロー
```
あなたのチームに合った使い方がある。
AgentPMは受託開発・Web制作・フリーランス・代理店、
それぞれの現場に合わせて使えるプロジェクト管理です。
```

### シーン1: 受託開発チーム (#outsourced)

タイトル: 要件定義から納品まで、発注者と一緒に管理

ストーリー:
```
Before:
- 要件定義はドキュメント共有、タスクはBacklog、進捗報告はExcel、承認はメール
- 発注者「今どこまで進んでるの？」→ PMがスクショ撮って報告
- 仕様変更の経緯が散逸。「言った言わない」で揉める

After (AgentPMで):
1. テンプレート「受託開発」を選択 → タスク・マイルストーン・Wiki・承認フローが自動生成
2. 開発者はCLIでタスク更新 → ポータルに自動反映
3. 発注者はポータルで進捗確認・バグ報告・見積もり承認
4. 仕様変更はWiki + レビュー承認で証跡が残る
5. ボール管理で「誰待ち」が常に明確
```

使う機能: CLI/スキル, ポータル, ボール管理, Wiki・レビュー承認, テンプレート

効果指標:
- プロジェクト立ち上げ: 2週間 → 当日
- 進捗報告作成: 週2時間 → 0（自動）
- 仕様変更の「言った言わない」: → ゼロ

### シーン2: Web制作会社 (#web)

タイトル: 企画から公開まで、複数案件を並行管理

ストーリー:
```
Before:
- 5案件を並行。各案件でBacklog + Chatwork + Googleドライブ
- ディレクターは毎朝全案件のステータスを確認するだけで1時間
- クライアントごとに報告フォーマットが違う

After (AgentPMで):
1. 案件ごとにスペース作成。テンプレート「Web制作」で即スタート
2. /project-status で全案件の状況を一括レポート
3. 各クライアントにポータルURL共有。報告不要
4. ボール一覧で「どの案件の何が止まっているか」が一目瞭然
```

使う機能: スペース管理, /project-status スキル, ポータル, ボール管理

効果指標:
- 朝の確認作業: 1時間 → 5分
- クライアント報告: 週5件 × 30分 → 0
- 案件の抜け漏れ: → ボールで可視化

### シーン3: フリーランスエンジニア (#freelance)

タイトル: 見積もり・進捗・請求を1ツールで完結

ストーリー:
```
Before:
- 見積もりはGoogleスプレッドシート、タスクはNotionかGitHub Issues
- 請求書は別ツール。工数の転記が毎月発生
- クライアントへの進捗報告はChatworkで手動

After (AgentPMで):
1. CLIでタスク作成・工数記録
2. 見積もりはポータルで送付 → ワンクリック承認
3. ポータルで進捗共有。報告メッセージ不要
4. 工数データをCSVエクスポート → 請求書作成
```

使う機能: CLI, ポータル, 見積もり・承認, CSVエクスポート

効果指標:
- 月末の請求作業: 半日 → 1時間
- 進捗報告の手間: → ゼロ
- 請求漏れ: → ゼロ

### シーン4: 代理店（制作会社管理）(#agency)

タイトル: 原価と売値を分けて、クライアントに安心を

ストーリー:
```
Before:
- 制作会社からの見積もりをExcelに転記 → マージン計算 → クライアント用清書
- 原価情報が漏れないか常にヒヤヒヤ
- 案件横断での粗利管理はさらに別のExcel

After (AgentPMで):
1. 制作会社がベンダーポータルから見積もり提出
2. 代理店がマージン設定 → クライアント向け売値が自動計算
3. クライアントにはポータルで売値のみ表示
4. 3段階承認フロー: ベンダー提出 → 代理店確認 → クライアント承認
```

使う機能: 代理店モード, ベンダーポータル, 3段階承認, 原価管理

効果指標:
- 見積もり作成: 2時間 → 15分
- 原価情報漏洩リスク: → ゼロ（システムで分離）
- マージン計算ミス: → ゼロ

### シーン5: SaaS開発チーム (#saas)

タイトル: スプリント管理 + ボールで、開発に集中

ストーリー:
```
Before:
- Jiraでスプリント管理。設定が複雑で、新メンバーのオンボーディングに時間がかかる
- プロダクトオーナーへの確認待ちタスクが誰にも見えない
- スタンドアップで「これ誰待ち？」が毎日発生

After (AgentPMで):
1. マイルストーンでスプリント管理
2. ボール管理でPO待ち・デザイン待ち・開発中が一目瞭然
3. CLIでタスク更新。管理画面を開かずに開発続行
4. /meeting-flow で朝会の議事録 → アクションアイテム自動起票
```

使う機能: マイルストーン, ボール管理, CLI, /meeting-flow スキル

効果指標:
- 「誰待ち？」の確認: → ボールで即座に判明
- スタンドアップの時間: 30分 → 15分
- 管理画面の操作: 日5-6回 → 0

## B. 導入事例 (/case-studies)

### ページ目的
実際の利用者の声で信頼性を担保。数値と具体的なエピソードで説得力を出す。

### ヒーロー
```
AgentPMを使っているチームの声

```

### 事例1: 株式会社クラフトテック（受託SaaS開発）

プロフィール:
- チーム: エンジニア8名 + PM2名
- 導入前: Backlog + Chatwork
- 利用: 3ヶ月目

見出し: 「管理画面を開くことがなくなった」

本文:
```
導入のきっかけ:
Claude Codeを使い始めて、コーディングはAIで加速した。でもタスク管理だけは
毎回Backlogを開いて手動更新。「ここもAIで回せないか」と探して見つけたのがAgentPM。

使い方:
開発者はCLIでタスク更新。PMは/project-statusで全体把握。
クライアントにはポータルURL共有して、進捗報告のChatworkが不要に。

効果:
- 管理画面の操作: 日5-6回 → 0回
- 進捗報告の作成: 週2時間 → 自動
- クライアントからの「今どうなってる？」: 激減
```

### 事例2: デジタルフロント合同会社（Web制作）

プロフィール:
- チーム: ディレクター3名 + デザイナー2名 + エンジニア5名
- 導入前: Backlog + Excel + メール
- 利用: 4ヶ月目
- 並行案件: 5件

見出し: 「進捗報告チャットが激減した」

本文:
```
導入のきっかけ:
5案件並行で、毎朝の確認だけで1時間。クライアントごとに報告フォーマットが違い、
Excel転記が負担だった。ポータルで報告不要になると聞いて導入。

使い方:
案件ごとにスペース作成。ボール管理で「どの案件の何が止まっているか」を把握。
クライアントにはポータルで進捗共有。

効果:
- 進捗問い合わせ: 週15回 → 3回（-80%）
- 朝の確認作業: 1時間 → ボール一覧を見るだけ
- 報告書作成: 完全不要に
```

### 事例3: 山本大輔（フリーランスエンジニア）

プロフィール:
- 1人で3案件管理
- 導入前: Notion + Googleスプレッドシート + Chatwork
- 利用: 2ヶ月目

見出し: 「請求漏れがゼロになった」

本文:
```
導入のきっかけ:
タスク管理、見積もり、請求、進捗報告がバラバラのツール。
月末に「あ、これ請求してなかった」が毎月あった。

使い方:
CLIでタスク管理。見積もりはポータルで送付→承認。
工数をCSVエクスポートして請求書作成。

効果:
- 請求漏れ: 月1-2件 → ゼロ
- 見積もり〜承認: メール3往復 → ワンクリック
- クライアント対応の手間: 半減
```

### 事例4: 株式会社メディアブリッジ（Web制作代理店）

プロフィール:
- 代理店: ディレクター4名
- 制作会社: 3社管理
- 導入前: Excel + メール
- 利用: 3ヶ月目

見出し: 「原価管理がExcelから一画面になった」

本文:
```
導入のきっかけ:

クライアントには売値のみ表示。

効果:
- 月末の請求作業: 2日 → 半日（-75%）
- マージン計算ミス: → ゼロ
- 原価漏洩リスク: → システムで完全分離
```
