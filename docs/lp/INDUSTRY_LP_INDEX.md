# 業種別LP 管理台帳

AI秘書（回収・催促・証跡）の業種別LPを一元管理する台帳。**新しいLPを作る前に必ずここに行を足し、公開したらステータスを更新する。**

- 配信の仕組み: `public/lpN/`（index.html + images/）を置くだけで `agentpm.app/lpN` で配信される（rewrite `/lp:id(\d+)`・middleware `STATIC_LP_PATTERN`）。**lp番号は本表で採番**し、欠番・重複を作らない。
- 作り方: **2026-07-12 全LPをエディトリアル広告版に刷新**。lp1（エディトリアル版）を正とし、`scripts/build-editorial-lps.py` がクラシック版辞書（`build-industry-lps.py` の INDUSTRIES）から語彙を自動変換して lp2〜lp8 を生成する（残存語彙チェック内蔵）。**新業種はクラシック辞書を1つ足して本スクリプトを実行**。デザイン原則は朱#E14A2B×クリーム×蛍光黄×ターコイズ・ロゴ風タイポ・「受領」印・写真は共有 `public/lp-assets/ed-*.jpg`。旧テンプレは scripts/lp1-classic-template.html / lp2-classic.html に凍結。
- 業種選定の3条件: **①毎月/毎年繰り返す ②相手のITリテラシーが低い ③遅れると法的・金銭的に痛い**（＋事務所/企業側に支払い能力があること）。

## 運用ルール

1. **採番は本表が正**。新規は候補表の上から順に検証し、公開したら「公開済み」表へ移す。
2. LPを公開したら: URL・公開日を記入し、`docs/lp/INDUSTRY_LP_INDEX.md` と memory（ai-secretary-pivot）を同時に更新する。
3. ヒアリング・広告で得た反応（問合せ数・CVR・ヒアリングメモ）は各行に追記していく。数字が付くまで優先順は仮置き。
4. フックの型は「**◯◯さえ、△△ば。**」で統一（シリーズとしてのブランド認知を作る）。
5. **問い合わせは全LP共通のステップフォーム**（1問ずつボタン選択→連絡先。相手件数・連絡手段・困りごとを聞き、回答は `lp_leads.extra` に保存）。設問の語彙は「やり取りする相手」など**業種非依存**で統一し、業種で限定する言い方（顧問先・クライアント等）を使わない。会社名欄のラベル/プレースホルダのみ業種別（生成スクリプトの `FORMCOMPANY`）。`source` はページのパスから自動判定するため、新LPでフォーム側の修正は不要。
