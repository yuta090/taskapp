## 解説キャラクター(2026-07-24決定)

**ガントとアイビー**(実在した二人の歴史人物)を、記事の対話形式の解説役として起用する。

- **役割分担(案)**:
  - **ガント**(ヘンリー・ガント/ガントチャート考案者): 計画・全体俯瞰・プロジェクト側の視点。
    「まず全体を見よう」と言う役
  - **アイビー**(アイビー・リー/「明日やる6つだけ書け」の人): 今日の実行・シンプルさ・
    タスク側の視点。「今日の6行から始めよう」と言う役
- **使いどころ**: 基礎講座の対話形式(初心者に概念を噛み砕く)、Q&Aの回答役、
  難所(ボール・見えない仕事など独自概念)の説明。全記事に出す義務はない
- **拡張**: 将来、パレート(優先順位)・デミング(振り返り)等を講師陣に追加できる設計
- **線引き(厳守)**: 史実(人物紹介・メソッドの由来)と、対話の演出(創作)を混ぜない。
  人物の史実を語るときは事実だけ、対話パートはキャラクターとしての演出と分かる形にする
- ビジュアル(イラスト)は別途。メディア名とは独立の決定(メディア名は上記候補から継続検討)

## 情報設計(IA)

トップは新着一覧ではなく「何を学びたいか」起点のハブ。

```
トップ(ハブ)
├─ 基礎講座        … 連載構造。初歩の初歩から順番に読める(E/F/G群)
├─ 業種別の実践    … 全業種×(人力のやり方+ツールのやり方)(B群)
├─ ツール活用      … ツール別使いこなし(Tシリーズ)+LINE/AI秘書(C/D群)
├─ よくある悩みQ&A … 実話台帳から起こした悩みと回答(検索KPI外)
├─ ユースケース百科 … Uシリーズ(事例交換プログラムによるインタビュー)
├─ テンプレ集      … Excel工程表・ガント・進捗管理表等の配布(中間CVエンジン)
└─ 用語辞典        … マイルストーン/クリティカルパス/PMBOK等(LLMO地層)
```

- 記事は日付でなく体系に配置。基礎講座は「次に読む」導線を持つ連載型
- 内部リンク: ニッチ記事(集客)→基礎講座(学び直し)→テンプレ/診断(中間CV)→agentpm/multica

### 表示速度の改善バックログ(page-perf-reviewer指摘・2026-07-24。公開前に別PRで)

1. **ISR化(最優先)**: 一覧・記事とも `force-dynamic` で毎アクセスDB往復。記事は
   `generateStaticParams`+`revalidate`、公開/更新時に `revalidatePath` で失効させる
2. **記事ページのクエリ二重発火**: generateMetadata と本体で `getPublishedPost` が2回走る。
   React の `cache()` でラップして1回に(低コスト・効果確実)
3. カバー画像を `next/image` 化(LCP対策)
4. 一覧 limit200 のページング(記事が増えたら)
