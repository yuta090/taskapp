# 販売SKU・価格の確定定義（現行実装ベース）

営業資料・商談で提示してよい**唯一のSKU**を、実装済みの範囲に固定する。
未実装の価格構造（定額＋グループ追加パック／Pro月額の具体額）を先出しして期待値を壊さないための正本。

正本コード: `src/lib/billing/entitlements.ts`（`PLAN_FEATURES` / `PLAN_LIMITS`）
課金実装: `src/app/api/stripe/*`（Stripe フラットサブスク）
GTM判断: 営業主導＋コンシェルジュ導入（`docs/ops/SHARED_LINE_BUDGET_RUNBOOK.md` 関連）

---

## 今日売れるSKU（これだけ）

**「フラット月額の Pro プラン ＋ 当社によるコンシェルジュ導入（開通代行）」** を直接営業で販売する。
課金は Stripe のサブスクリプション（`checkout → webhook → org_billing → 402ゲート`）で自己完結している。

- プランは **free / pro / enterprise** の3段。**従量・追加パックのSKUは無い**（フラットサブスクのみ）。
- 価格 IDは環境変数 / Stripe ダッシュボード定義。**コードに金額は無い**。

## 商談で言ってはいけないこと（未実装・期待値破壊）

- ❌ **「定額＋グループ追加パック」**：パックSKUのメータリングは未実装。フラット月額でしか請求できない。
- ❌ **「Pro は月◯万円（10万円未満）」等の具体額**：LLM抽出原価とLINE規約の複数アカウント可否が未確定。価格は**実測後に確定**。資料に数字を刷らない。
- ❌ **「無料で登録すればすぐ自動タスク化される」**：共通Botのプロビジョニングと AI設定は当社作業。**セルフサーブでアハ体験に到達しない**（`activation` 系修正で文言は申込制に是正済み）。
- ❌ **「自社LINEは即日開通」**：自社LINE（`owner_type='org'`）の開通は手動オペ。**納期を提示する**（例:「お申込みから◯営業日で開通」）。同時受注上限も設ける。

## 提示フォーマット（1枚に固定）

1. 月額（Pro フラット・当面は数値を口頭/都度見積、資料には刷らない）
2. 導入支援（コンシェルジュ開通）の有無と料金
3. 対象LINEグループ数 / 共通LINE送信上限（上表の値）
4. 自社LINE開通の標準納期
5. 上限超過時の扱い（縮退/停止・`SHARED_LINE_BUDGET_RUNBOOK` 準拠）

## 将来（今は着手しない）

従量メータリング＋グループ追加パックのSKU化は、価格実測が済んでから別プロジェクトで行う。
それまでは本定義（フラット月額＋手動調整）を唯一の販売単位とする。
