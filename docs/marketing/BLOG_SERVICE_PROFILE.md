# ブログ記事のサービスプロファイル（CTA原本）

`japanese-blog-writing` スキルが記事のCTA・トンマナを組み立てる際に参照する正本。
記事を書く前に必ずここを読み、**誇大表示ガード**（末尾）に違反しないか確認する。

## サービス概要

| 項目 | 内容 |
|------|------|
| プロダクト | AI秘書（回収・催促・証跡エンジン） |
| 売り方 | SaaS導入ではなく **「AI秘書を雇う」** |
| 正体 | 汎用PMツールではない。**資料・回答・お金の「回収」を代行する** |
| 最初のターゲット | 税理士・会計事務所（顧問先からの資料回収） |
| 次の候補 | 建設業（下請けの安全書類回収） |
| 対応チャネル | LINE / メール / Chatwork / Slack / Google Chat |
| 親ブランド | skara.co.jp（傘） |

## 価値提案（記事内で使う言い回し）

- **削減**: 資料回収は「催促の連絡」だけではない。未着リストアップ → 個別連絡 → 中身確認 → 不足の再依頼 → 受領記録 → 再催促、の一連。1社あたり月30分 × 50社 = **月25時間 ≒ 月4.5万円分**の作業
- **増収**: 回収が回れば担当社数を増やせる。**＋5社 × 顧問料月3万 = 月15万の余地**（採用難への答え）
- **アンカー**: 「顧問先1社分の顧問料以下で雇える」
- 口調は**詰めない・責めない**。AIであることは名乗る。名義は事務所（白ラベル）

## 読者と検討段階 → CTAの出し分け

| 記事タイプ | 想定読者 | CTA |
|-----------|---------|-----|
| Know（資料回収の悩み・繁忙期・業務効率化の一般論） | 潜在層 | 診断（multica）／関連ガイド・チェックリスト |
| Do / 比較（回収方法の比較、ツール比較、LINE活用） | 準顕在層 | サービス資料・LP誘導 |
| Buy（導入検討・料金・事例） | 顕在層 | 無料相談・パイロット申込 |

### 入り口は「痛み別」に分離する（統合しない）

- **multica**: タスク滞留の診断 → 商談 → パイロット（skara.co.jp/shindan）。**課題の仕分け機（ルーター）**として使う
- **agentpm（AI秘書）**: 回収の痛みが明確な読者向け。税理士LPへ

記事が扱った悩みに応じてどちらかへ送る。両方を並べて「お好きな方へ」はしない。

## CTA文例（文中・宣伝臭を出さない書き方）

> この工程を人がやると1社あたり月30分前後かかる。筆者らが開発しているAI秘書は、この「未着の把握 → 個別連絡 → 受領記録」の往復をLINEやメール上で肩代わりする設計になっている。（→ リンク）

記事末尾は、その記事の悩みに対応する入口を1つだけ、オファーを明示して置く。

## 誇大表示ガード（厳守）

公式HPで**未実装機能の販売表示**が問題として検出された経緯がある。記事でも以下を書かない:

- **未実装の機能を実装済みとして書かない**（例: Chatwork/Google Chat秘書、SSO/SAML、AES-256などは実装状況を確認してから）
- **根拠のない数値・実績を書かない**（「導入企業◯社」「削減率◯%」は実データがある場合のみ）
- **「5社限定」等の煽り**は使わない
- 上記の「月4.5万円分」「月15万の余地」は**試算である旨を明記**して使う（実績値ではない）
- 料金・返金条件に触れる場合は、特商法ページ・pricingページと矛盾しないか確認する

不明な点は書かずに、ユーザー（高橋）に確認する。
