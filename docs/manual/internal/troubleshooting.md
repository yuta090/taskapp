# トラブルシューティング

よくあるトラブルと対処方法をまとめています。

---

## 招待メールが届かない

**症状**: メンバーやクライアントに招待メールを送信したが、相手に届いていない。

**原因**:
- 迷惑メール（スパム）フォルダに振り分けられている
- メールアドレスの入力ミス
- 受信側のドメイン制限やフィルタリング設定

**対処法**:
1. 受信者に迷惑メールフォルダを確認してもらう
2. 入力したメールアドレスが正しいか確認する
3. 企業のメールサーバーで外部メールがブロックされていないか、IT担当者に確認を依頼する
4. 上記で解決しない場合、別のメールアドレスで再度招待を送信する

---

## 招待リンクの有効期限切れ

**症状**: 招待メール内のリンクをクリックしても「リンクが無効です」と表示される。

**原因**:
- 招待リンクの有効期限（30日間）が切れている

**対処法**:
1. プロジェクト設定 > メンバーセクションを開く
2. 該当メンバーの招待を一度削除する
3. 同じメールアドレスで再度招待を送信する
4. 受信者に新しいメール内のリンクからアクセスしてもらう

> **注意**: クライアントの場合も同様の手順で再招待できます。ロールに `client` を選択してください。

---

## 権限不足エラー

**症状**: 操作しようとすると「権限がありません」と表示される。

**原因**:
- 現在のロール（member / client）では許可されていない操作を実行しようとしている
- プロジェクトレベルの権限が不足している

**対処法**:
1. 自分のロールを確認する（設定 > メンバーセクション）
2. 必要な権限を持つロール（admin 以上）を確認する
3. プロジェクトの admin または組織の owner にロール変更を依頼する

以下は主なロール別の権限範囲です。

| 操作 | admin | member | client |
|------|-------|--------|--------|
| メンバー招待 | 可 | 不可 | 不可 |
| 設定変更 | 可 | 閲覧のみ | 不可 |
| タスク作成・編集 | 可 | 可 | 不可 |
| コメント | 可 | 可 | 可 |

## 通知が届かない

**症状**: タスクの更新やレビュー依頼の通知が届かない。

**原因**:
- アプリ内通知は確認しているが、メールや Slack の通知が設定されていない
- Slack 連携が切断されている
- メール通知がオフになっている

**対処法**:
1. **アプリ内通知の確認**: 画面右上のベルアイコンをクリックして通知一覧を確認する
2. **メール通知**: 設定画面でメール通知が有効になっているか確認する
3. **Slack 通知**: 設定 > Slack 連携セクションで接続状態を確認する
   - 「接続済み」になっているか
   - プロジェクトにチャンネルがリンクされているか
   - 通知タイプのチェックが有効か
4. Slack 接続が切れている場合は再接続する

## Google Calendar 接続エラー

**症状**: Google カレンダーとの接続に失敗する、または接続が切れている。

**原因**:
- OAuth トークンの有効期限切れ
- Google アカウントのパスワード変更やセキュリティ設定の変更
- 必要なスコープ（カレンダー読み取り権限）が許可されていない

**対処法**:
1. プロジェクト設定 > Google カレンダーセクションを開く
2. 接続状態を確認する
   - **期限切れ**（赤バッジ）の場合: 「再接続」ボタンをクリックする
   - **未接続**（グレーバッジ）の場合: 「Google カレンダーに接続」をクリックする
3. Google の認証画面で以下を確認する
   - 正しい Google アカウントでログインしているか
   - 「カレンダーの表示」権限を許可しているか
4. 接続完了後、接続状態が「接続済み」（緑バッジ）になっていることを確認する

> **ヒント**: 社内のセキュリティポリシーで OAuth が制限されている場合は、IT 管理者に TaskApp の OAuth アプリを許可リストに追加してもらう必要があります。

## 日程確定ができない

**症状**: 日程調整の提案で「確定」ボタンが押せない、またはグレーアウトしている。

**原因**:
- 全参加者の回答が揃っていない
- すべてのスロットで「参加不可」の回答がある（確定可能なスロットがない）
- 提案の有効期限が切れている（ステータスが `expired`）
- 確定権限を持っていない（提案作成者またはスペース admin のみ確定可能）

**対処法**:
1. **未回答者がいる場合**: 回答状況マトリクスで未回答の参加者を確認し、個別に連絡する
2. **確定可能なスロットがない場合**: 提案をキャンセルし、新しい候補日時で再提案する
3. **有効期限切れの場合**: 新しい提案を作成する（期限切れの提案は確定できません）
4. **権限の問題**: 提案の作成者またはスペースの admin に確定を依頼する

## 議事録パーサーのエラー

**症状**: 議事録に SPEC 記法を書いたが、タスクが自動生成されない、またはエラーが表示される。

**原因**:
- SPEC 記法の書式が正しくない

**対処法**:

正しい SPEC 記法を確認してください。

```
正しい例:
- [ ] SPEC(/spec/design#header): ヘッダーデザインの決定

よくある間違い:
- [ ] SPEC /spec/design#header: ヘッダーデザイン    ← () が抜けている
- [ ] SPEC(/spec/design#header) ヘッダーデザイン    ← : が抜けている
- [ ] spec(/spec/design#header): ヘッダーデザイン   ← SPEC は大文字
- [x] SPEC(/spec/design#header): ヘッダーデザイン   ← チェック済みは無効
```

### 記法のルール

| 要素 | 必須 | 説明 |
|------|------|------|
| `- [ ]` | 必須 | 未チェックのリスト項目 |
| `SPEC` | 必須 | 大文字で記述 |
| `(/path#anchor)` | 必須 | 仕様書パスとアンカー（括弧必須） |
| `:` | 必須 | パスの後にコロン |
| タイトル | 必須 | コロンの後にスペースを空けてタイトル |

## ボールが戻せない

**症状**: ボールを `client` から `internal`（またはその逆）に変更できない。

**原因**:
- タスクのステータスが `done` または `closed` になっている（完了済みタスクはボールを変更できません）
- 権限が不足している（client ロールではボールを変更できません）

**対処法**:
1. タスクのステータスを確認する
   - `done` / `closed` の場合: まずステータスを `in_progress` に戻してからボールを変更する
2. client ロールの場合: ボールの変更は開発チームに依頼する（コメントで連絡）
3. それでも変更できない場合: ブラウザを再読み込みして再試行する

## ページが表示されない

**症状**: 画面が白くなる、エラーが表示される、またはレイアウトが崩れる。

**原因**:
- ブラウザのキャッシュが古い
- 推奨されていないブラウザを使用している
- ネットワーク接続の問題

**対処法**:
1. **ブラウザキャッシュのクリア**:
   - Chrome: `Ctrl + Shift + Delete`（Mac: `Cmd + Shift + Delete`）
   - Safari: メニュー > 履歴 > 履歴を消去
   - Edge: `Ctrl + Shift + Delete`
2. **推奨ブラウザ**で開き直す:
   - Google Chrome（最新版）
   - Microsoft Edge（最新版）
   - Safari（最新版）
   - Firefox（最新版）
3. **ネットワーク確認**: 安定したインターネット接続があるか確認する
4. **シークレットモード**: ブラウザのシークレット（プライベート）モードで開いて問題が再現するか確認する

> **ヒント**: シークレットモードで正常に表示される場合、ブラウザの拡張機能が干渉している可能性があります。拡張機能を無効にして再試行してください。
