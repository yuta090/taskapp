## クイックリファレンス

| 操作 | 方法 |
|------|------|
| タスク作成 | 右下の「+」ボタン |
| タスク詳細 | タスク行をクリック（右ペインに表示） |
| インスペクター閉じる | `Esc` キー |
| ボールパス | インスペクター内のボール切替 |
| 会議開始 | 会議詳細 → 「開始」ボタン |
| 議事録からタスク生成 | `- [ ] SPEC(/spec/path#anchor): タイトル` 記法 |
| フィルター | タスク一覧上部のフィルターバー |

## 基本概念

### ボール（Ball）
「次に誰がアクションするか」を明示する仕組みです。

| ボール | 意味 |
|--------|------|
| `internal`（社内） | 開発チームが対応中 |
| `client`（クライアント） | クライアントの確認・回答待ち |

### タスクタイプ
| タイプ | 説明 |
|--------|------|
| `task` | 通常のタスク |
| `spec` | 仕様タスク（spec_path と decision_state が必要） |

### Amber-500 バッジ
黄色（Amber-500）のバッジが付いた要素は、クライアントポータルにも表示される項目です。
