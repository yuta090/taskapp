# 用語集

TaskApp で使用される専門用語の一覧です。

---

## 基本概念

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **ボール (ball)** | 「次にアクションすべき側」を示す概念。`client`（外部 = クライアント側）または `internal`（社内側）のいずれか。UI上では「外部」「社内」と表示される | タスク一覧、インスペクター、ダッシュボード |
| **ボールパス** | ボールの所有を相手側に移すこと。例: 社内で作業完了後、外部に確認依頼としてボールを渡す | インスペクター内のボール切替操作 |
| **オリジン (origin)** | タスクを最初に起票した側。`client`（外部起票）または `internal`（社内起票） | タスク作成時に自動設定 |

---

## タスク関連

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **タスクタイプ** | タスクの種類。`task`（通常タスク）と `spec`（仕様決定タスク）の2種類 | タスク作成時に選択 |
| **仕様タスク (spec)** | 仕様に関する意思決定を追跡するタスク。`spec_path`（仕様書パス）と `decision_state`（決定状態）が必須 | 仕様決定が必要な場面 |
| **決定状態 (decision_state)** | 仕様タスクの決定プロセスの状態。`considering`（検討中）→ `decided`（決定済み）→ `implemented`（実装済み） | 仕様タスクのインスペクター |
| **ステータス** | タスクの進行状態。`todo` → `in_progress` → `in_review` → `done` → `closed` の順に遷移 | タスク一覧、インスペクター |
| **担当者 (assignee)** | 現在タスクを担当している人。1タスクにつき1人 | タスク作成・編集時 |
| **責任者 (owners)** | タスクの意思決定者・通知を受ける関係者。複数人設定可能 | タスク作成・編集時（表示設定で非表示にも可能） |

## UI 関連

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **3ペインレイアウト** | 画面を3つの領域に分割した固定レイアウト。左ナビ(240px) + メインエリア(flex-1) + インスペクター(400px) | TaskApp のすべての画面 |
| **インスペクター** | 画面右側の詳細パネル（幅 400px）。タスクや会議の詳細情報を表示する。メインエリアに重ならず常に並列表示 | タスク・会議・レビューの詳細表示 |
| **Amber-500 バッジ** | 黄色（Amber-500）のバッジ。クライアントポータルにも表示される要素であることを示す目印 | タスク一覧、インスペクター |

---

## レビュー関連

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **レビューステータス** | レビューの状態。`pending`（待ち）→ `approved`（承認）/ `changes_requested`（変更依頼）/ `rejected`（却下） | レビューセクション |
| **必須レビュアー** | 指名されたレビュアー。全員が承認しないとレビュー完了にならない | レビュー作成時に指名 |
| **任意レビュー** | レビュアーを指名しない形式。プロジェクトメンバーの誰でもレビュー可能 | 軽微な確認事項 |
| **監査証跡** | レビューの承認・変更依頼・却下の記録。レビュアー名、タイムスタンプ、関連会議IDが自動記録される | レビュー履歴の確認 |

---

## 会議・日程調整

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **日程提案 (Scheduling Proposal)** | 会議の候補日時を提案し、参加者の回答を集約する機能 | 会議一覧 > 日程調整 |
| **スロット** | 日程提案における候補日時の1つ。1提案につき2〜5個設定可能 | 日程提案の作成・回答 |
| **FreeBusy** | Google カレンダー連携時に表示される参加者の空き状況。空き（緑）/ 予定あり（赤） | 日程提案の候補日時設定 |
| **SPEC 記法** | 議事録内でタスクを自動生成するための記法。`- [ ] SPEC(/path#anchor): タイトル` の形式 | 議事録の記入 |

---

## 技術用語

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **RPC 関数** | サーバー側で実行されるリモートプロシージャコール関数。主な関数: `rpc_pass_ball`（ボールパス）、`rpc_decide_considering`（仕様決定）など | 内部実装・MCP ツール |
| **クライアントポータル** | 外部（クライアント）専用のアクセス画面（`/portal`）。トークンベース認証でアカウント作成不要 | 外部ユーザーのアクセス |
| **MCP (Model Context Protocol)** | AI ツールから TaskApp を操作するためのプロトコル。40以上のツールを提供 | AI 連携・自動化 |

---

## ロール・権限

| 用語 | 説明 | 使用場面 |
|------|------|---------|
| **owner** | 組織の所有者。全権限を持つ。1組織につき1名 | 組織管理 |
| **admin** | プロジェクト管理者。設定変更、メンバー招待が可能 | プロジェクト設定 |
| **member** | プロジェクトメンバー。タスク・会議の操作が可能 | 日常のタスク操作 |
| **client** | クライアント。ポータル経由で閲覧・コメント・レビュー | クライアントポータル |
