# Slack連携のはじめかた

AgentPM と Slack を接続すると、タスクの更新通知を Slack チャンネルで受け取ったり、Slack からタスクを作成したりできます。このページでは、組織管理者が最初に行う接続設定から、日常での使いかたまでを説明します。

## できること

| 機能 | 説明 |
|------|------|
| チャンネル通知 | タスクの作成・更新・承認などのイベントを、プロジェクトに紐づけたチャンネルへ自動投稿 |
| `/agentpm` コマンド | Slack から直接タスクを作成（タスク作成モーダルが開きます） |
| @AgentPM メンション | チャンネルで Bot にメンションすると、AI がプロジェクトの状況を踏まえて応答（AI設定が必要） |

## 前提条件

- AgentPM の**組織オーナーまたは管理者**であること（メンバー・クライアント権限では接続できません）
- 接続先 Slack ワークスペースで**アプリのインストールを許可できる権限**があること

## Step 1: 組織と Slack ワークスペースを接続する

組織単位の接続は最初に1回だけ行います。

1. 左サイドバー上部の**組織名**をクリックし、メニューから**「外部連携」**を選択する
2. Slack セクションの**「Slackと連携する」**ボタンをクリックする
3. Slack の許可画面が開くので、接続先ワークスペースを確認して**「許可する」**をクリックする
4. AgentPM に戻り、「Slackワークスペースを接続しました」と表示されれば完了。ワークスペース名が表示されます

> **接続に失敗する場合**: 「Slack連携に失敗しました」と表示されたときは、時間をおいて再試行してください。ボタン自体が表示されない場合は、システム側で Slack 連携が有効化されていません（運用者向けの [Slack App セットアップ手順](../../SLACK_SETUP.md) を参照）。

## Step 2: プロジェクトとチャンネルを紐づける

通知はプロジェクト（Space）単位で、リンクしたチャンネルに送信されます。

1. **通知を受け取りたい Slack チャンネルで、Bot を招待する**:

   ```
   /invite @AgentPM
   ```

   ※ プライベートチャンネルでは、この招待を先に行わないと AgentPM 側にチャンネルが表示されません。

2. AgentPM で対象プロジェクトの**プロジェクト設定 → 外部連携 → Slack** を開く
3. チャンネル一覧から対象チャンネルを選び、**「リンク」**をクリックする
4. 通知タイプ（タスク作成・更新・承認など）はチャンネルごとにチェックボックスで調整できます

詳しい画面操作は [プロジェクト設定 › Slack連携](./settings.md#slack連携) を参照してください。

## 使いかた

### タスク通知を受け取る

チャンネルをリンクすると、選択した通知タイプのイベントが自動で投稿されます。追加の操作は不要です。

### Slack からタスクを作成する（`/agentpm`）

リンク済みのチャンネルで次のコマンドを入力すると、タスク作成モーダルが開きます:

```
/agentpm
```

- 「このチャンネルはAgentPMに連携されていません」と表示された場合は、Step 2 のチャンネルリンクを先に行ってください

### AI アシスタントに質問する（@メンション）

リンク済みのチャンネルで Bot にメンションすると、AI がプロジェクトの文脈を踏まえて応答します:

```
@AgentPM 今週の確認待ちタスクを教えて
```

- 「AI機能が設定されていません」と表示された場合は、組織の**外部連携 → AI設定**で AI プロバイダーの API キーを登録してください

## 接続を解除する

- **チャンネル単位**: プロジェクト設定 → 外部連携 → Slack で対象チャンネルのリンクを解除
- **組織単位（全解除）**: 組織メニュー → 外部連携 → Slack の**「接続を解除」**。確認ダイアログで「解除」を選ぶと、全プロジェクトのチャンネルリンクがまとめて解除されます

## トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| 「Slackと連携する」ボタンが表示されない | システム側で Slack 連携が無効。運用者に連絡（環境変数 `NEXT_PUBLIC_SLACK_ENABLED` と Slack App 設定が必要） |
| チャンネル一覧に目的のチャンネルが出ない | プライベートチャンネルは `/invite @AgentPM` で Bot を招待してから再読み込み |
| `/agentpm` が「連携されていません」と返す | そのチャンネルがどのプロジェクトにもリンクされていない。Step 2 を実施 |
| 通知が届かない | ①チャンネルのリンク状態 ②通知タイプのチェック ③Bot がチャンネルに参加しているか、の順に確認 |
| @メンションに反応しない | チャンネル未リンク、または AI 設定（API キー）未登録 |
