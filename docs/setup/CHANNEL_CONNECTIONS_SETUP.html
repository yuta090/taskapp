AI秘書 チャネル接続セットアップ手順

秘書の「繋ぐ先」を主要チャットへ拡張するために、あなた（運用者）が各サービス側で設定して発行する値と、それを TaskApp のどこに登録するかをまとめた手順書です。

生成日: 2026-07-20 ／ 対象コード: src/lib/channels/registry.ts・src/lib/channels/adapters/

読み方 — 各チャネルの「発行する資格情報」の表で この列 の値をコピーし、TaskApp の秘書コンソール（または channel_accounts）へ貼り付けます。機密 は暗号化保存され、UIでは再表示されません。実装状況バッジ（下記）で「今すぐ使える範囲」を確認してください。

0. 共通の前提（最初に一度だけ）

個々のチャネル設定の前に、サーバー側で以下がそろっている必要があります。

項目値・場所用途
SYSTEM_ENCRYPTION_KEY（環境変数）32バイト以上のランダム文字列チャネルの資格情報を channel_accounts.credentials_encrypted に暗号化保存する鍵。未設定だと登録・送信が動きません。
公開URL（Webhookベース）本番デプロイのHTTPSドメイン受信Webhookを各サービスに登録する際のベースURL。例: https://app.example.com
プラン設定org_billing.plan_id白ラベル・即時通知・1:1DM は Pro 専有（src/lib/billing/entitlements.ts）。

凡例（実装状況）: GA 送受信とも利用可 BETA 送信は実装済み・受信は準備中 PLANNED 登録のみ（ロードマップ） PRO 上位プラン専有

1. LINE

GA

送信・受信対応 / グループ・1:1DM可 / 署名: HMAC-SHA256 ／ 開発者コンソール: developers.line.biz

  1. LINE Developers で Messaging APIチャネルを作成（共通LINEは既存の共有アカウントを使用）。
  2. 「Messaging API設定」で チャネルアクセストークン（長期）を発行。
  3. 「チャネル基本設定」で チャネルシークレットを控える。
  4. Webhook URL に {公開URL}/api/channels/line/webhook を登録し、Webhookの利用=オン・応答メッセージ=オフ。
  5. 自社LINE（白ラベル）は Pro のみ。共通LINEはプラットフォーム共有アカウントを使用。
発行する値貼り付け先キー区分
Channel secretchannel_secret機密
Channel access token（長期）access_token機密

送信先(to): LINE userId（Uで始まる）／グループID（Cで始まる）。PRO 自社名義・即時通知・1:1DM。

2. Slack

GA

送信・受信対応 / チャンネル・DM可 / 署名: HMAC-SHA256 ／ 管理: api.slack.com/apps

  1. 「Create New App」→ From scratch でアプリ作成。
  2. 「OAuth & Permissions」で Bot Token Scopes に chat:write・channels:read・groups:read を追加。
  3. ワークスペースにインストールし Bot User OAuth Token（xoxb-…）を取得。
  4. 「Basic Information」→ App Credentials の Signing Secret を控える。
  5. 受信を使う場合は「Event Subscriptions」で Request URL に {公開URL}/api/slack/webhook を登録。
  6. 秘書に投稿させたい各チャンネルで /invite @アプリ名（未参加だと not_in_channel）。
発行する値貼り付け先キー区分
Bot User OAuth Token（xoxb-）bot_token機密
Signing Secretsigning_secret機密

送信先(to): チャンネルID（Cで始まる）／ユーザーID（U・Wで始まる）。

3. Chatwork

BETA

送信対応（受信は準備中）/ ルーム・DM可 ／ トークン発行: API設定

  1. Chatworkにログインし「サービス連携 → API」から APIトークンを発行。
  2. 秘書に投稿させたいルームを開き、URLの rid 値（数字）＝ room_id を控える。
  3. （受信対応時）Webhook Token を発行し webhook_token に登録。
発行する値貼り付け先キー区分
API Tokenapi_token機密
Webhook Token（受信対応時）webhook_token機密

送信先(to): ルームID（room_id・数字のみ）。日本のSMBで普及。

4. Google Chat

BETA

送信対応（Incoming Webhook方式）/ スペース宛 ／ ドキュメント: Webhooks

  1. Google Chat で通知したいスペースを開く（Webhookはスペースにのみ設定可）。
  2. スペース名 → 「アプリと連携」→「Webhook」→「Webhookを追加」で名前を付けて作成。
  3. 発行された Webhook URL（chat.googleapis.com/…）をコピー。
発行する値貼り付け先キー区分
Incoming Webhook URLwebhook_url機密

双方向（受信）にはChat API＋サービスアカウントが別途必要。送信のみ本方式。

5. Discord

BETA

送信対応（チャンネルWebhook）/ チャンネル宛 ／ 開発者ポータル: Developer Portal

  1. 通知したいチャンネルの「⚙ 編集 → 連携サービス → ウェブフック」を開く。
  2. 「新しいウェブフック」を作成し、名前とアイコンを設定。
  3. 「ウェブフックURLをコピー」で Webhook URL（discord.com/api/webhooks/…）を取得。
発行する値貼り付け先キー区分
Channel Webhook URLwebhook_url機密

Bot（受信・スラッシュコマンド・Ed25519署名）は後続。まずはチャンネル送信。

6. Telegram

BETA

送信対応 / グループ・1:1DM可 ／ 作成: @BotFather

  1. Telegramで @BotFather に /newbot を送り、名前とユーザー名を設定。
  2. 発行された Bot Token（123456:AA…）を控える。
  3. 送信先の chat_id を取得（対象チャット／グループにBotを追加後、getUpdates 等で確認）。
  4. （受信対応時）setWebhook に {公開URL}/api/channels/telegram/webhook と secret_token を設定。
発行する値貼り付け先キー区分
Bot Tokenbot_token機密
Webhook Secret Token（受信対応時）webhook_secret機密

送信先(to): chat_id（数値。ユーザー／グループ／チャンネル）。

7. Microsoft Teams

BETAPRO

送信対応（Incoming Webhook / Workflows）/ チャンネル宛 ／ 手順: Incoming Webhook

  1. 通知したいチームのチャンネル「…」→「ワークフロー」（または従来のコネクタ）を開く。
  2. 「Webhookを使用してチャネルに投稿する」テンプレートを追加し、対象チャネルを選択。
  3. 発行された Webhook URL（*.webhook.office.com または *.logic.azure.com）をコピー。
発行する値貼り付け先キー区分
Incoming Webhook URLwebhook_url機密

Adaptive Cardで送信。双方向はBot Framework＋Azureアプリ登録が別途必要。

8. WhatsApp Business

BETAPRO

送信対応（Meta Cloud API）/ 1:1DM ／ ドキュメント: Cloud API

  1. Meta for Developers でアプリを作成し WhatsApp プロダクトを追加。
  2. Meta Business アカウントの認証・電話番号の登録を完了（審査あり）。
  3. System User Access Token と Phone Number ID を取得。
  4. （受信対応時）Webhookに {公開URL}/api/channels/whatsapp/webhook を登録し App Secret で X-Hub-Signature-256 を検証。
発行する値貼り付け先キー区分
System User Access Tokenaccess_token機密
Phone Number IDphone_number_id公開可
App Secret（受信対応時）app_secret機密

24時間ウィンドウ外はテンプレートメッセージのみ（v1は本文テキスト対応）。ビジネス認証が前提。

9. Facebook Messenger

PLANNEDPRO

ロードマップ（アダプタ未実装）/ 1:1DM ／ ドキュメント: Messenger Platform

現状は登録枠のみ。利用にはFacebookページ・アプリのMetaレビューが必要です。準備する値は Page Access Token・App Secret（page_access_token / app_secret）。実装完了までは設定不要です。

やることチェックリスト

今すぐ繋ぐなら GA/BETA のチャネルから。あなた側の作業だけを抜き出すと以下です。

  • 共通: 本番に SYSTEM_ENCRYPTION_KEY を設定し、公開HTTPSドメインを確定する。
  • LINE: Messaging APIチャネルで channel secret / access token を発行し、Webhook URL を登録する。
  • Slack: アプリを作り chat:write 等のscopeを付与、xoxb トークンと Signing Secret を取得、対象チャンネルにBotを招待する。
  • Chatwork: APIトークンを発行し、投稿先ルームの room_id（数字）を控える。
  • Google Chat: 対象スペースで Incoming Webhook を作成し URL を控える。
  • Discord: 対象チャンネルで Webhook を作成し URL を控える。
  • Telegram: @BotFather でBotを作成し token を取得、送信先 chat_id を控える。
  • Microsoft Teams（Pro）: 対象チャネルの Workflows/コネクタで Incoming Webhook URL を発行する。
  • WhatsApp Business（Pro）: Metaでビジネス認証・番号登録を済ませ、access token と phone_number_id を取得する。
  • Messenger: 実装完了まで作業不要（ロードマップ）。
実装メモ（開発者向け） — 送信の追加は完了しています（src/lib/channels/adapters/、統一入口は deliverToChannel(channel, ctx)）。受信Webhookは各サービスで署名方式が異なり、マルチテナントの資格情報照合を伴うため、チャネルごとに個別PRで追加します（LINEは実装済み）。本手順書の「受信対応時」項目は、その受信を有効化する際に必要になる値です。
この手順書は src/lib/channels/registry.ts の定義（表示順・資格情報キー・実装状況・Pro区分）に対応しています。チャネルを追加・変更したら、レジストリと本HTMLの両方を更新してください。
